# Security Policy

## Reporting a Vulnerability

The CRI-O credential provider project follows the same security policy as the
main [CRI-O](https://github.com/cri-o/cri-o) project.

To report a security vulnerability, please use the
[GitHub Security Advisory](https://github.com/cri-o/crio-credential-provider/security/advisories/new)
feature to privately report the issue.

You can also report vulnerabilities by following the
[Kubernetes Security and Disclosure Information](https://kubernetes.io/docs/reference/issues-security/security/#report-a-vulnerability)
process.

## Response Time

The maintainers will acknowledge vulnerability reports within 7 days and aim to
provide a fix or mitigation plan within 30 days.

## Supported Versions

Security fixes are applied to the most recent tagged release (the highest
semver tag in the repository).

| Version                    | Supported          |
| -------------------------- | ------------------ |
| Most recent tagged release | :white_check_mark: |
| Older releases             | :x:                |

## Security Announcements

Security announcements are made through
[GitHub Security Advisories](https://github.com/cri-o/crio-credential-provider/security/advisories).
